postprocess_scope_learning_results(source_filename, target_filename)

source code 

Dado un archivo resultado de un aprendizaje, transforma las dos columnas finales (que se asume son un scope original y otro aprendido, ambos en formato FOL) y le aplica las reglas de morante2009 y lo transforma a formato BIO. Asume que los primeros cuatro atributos del archivos son: document, sentence, num_token, hc_start

Parameters:
  • source_filename (string) - nombre del archivo con el resultado del aprendizaje
  • target_filename (string) - nombre del archivo con el resultado del procesamiento

gen_conll_file_scope(dbname, tablename, sentence_type, filename, xs, y, predicted_y)

source code 

Genera el archivo para el entrenamiento/evaluación de scopes con CRF++, a partir de la tabla de bioscope que se le indique (bioscope80_scope o bioscope20_scope) Este archivo está en formato CoNLL, tiene una línea por token, los atributos están separados por espacio, y el último es el que vamos a usar para clasificar. Las oraciones está­n separadas por líneas en blanco

Parameters:
  • dbname (string) - nombre del archivo que tiene la base de datos
  • tablename (string) - nombre de la tabla a partir de la cual generar el archivo
  • sentence_type (string) - indica el string que deben tener las instancias en el campo SENTENCE_TYPE (solo tiene sentido si has_instances=False
  • xs (List) - lista de atributos a generar. Tienen que ser iguales a las columnas de la tabla de atributos de bioscope. No incluyen la clase a aprender.
  • y (List) - Clase a aprender (es uno de los atributos)
  • predicted_y (string) - Ya me pasan la clase aprendida (estoy generando directamente el archivo de evaluación)

gen_conll_file_hc(dbname, tablename, sentence_type, filename, xs, y, predicted_y)

source code 

Genera el archivo para el entrenamiento/evaluación con CRF++, a partir de la tabla de bioscope que se le indique Este archivo está en formato CoNLL, tiene una línea por token, los atributos están separados por espacio, y el último es el que vamos a usar para clasificar. Las oraciones están separadas por líneas en blanco

Parameters:
  • dbname (string) - nombre del archivo que tiene la base de datos
  • tablename (string) - nombre de la tabla a partir de la cual generar el archivo
  • sentence_type (string) - indica el string que deben tener las instancias en el campo SENTENCE_TYPE. Si vale ALL, entonces genera sobre todas las tuplas
  • xs (List) - lista de atributos a generar. Tienen que ser iguales a las columnas de la tabla de atributos de bioscope. No incluyen la clase a aprender.
  • y (List) - Clase a aprender (es uno de los atributos)
  • predicted_y (string) - Ya me pasan la clase aprendida (estoy generando directamente el archivo de evaluación)
